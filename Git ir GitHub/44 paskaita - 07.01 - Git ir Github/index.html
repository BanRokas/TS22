Git ir Github

Git'as - populiariausia kodo versijavimo programa.
GitHub'as - vienas populiaresnių versijuojamo kodo talpinimo platformų.

Git komandos

  • init - sukuriama git repozitorija dabartiniame aplankale
  • status - nurodo informaciją apie lokalios git'o repozitorijos statusą
  • add - prideda nurodytus failus prie commit'inimui paruoštų failų (stage'ina)
  • rm --cached || restore --staged - išima nurodytus failus iš stage'inimo zonos (unstage'ina)
  • commit -m 'zinute' - sukuria kodo versiją iš stage'intų failų ir prideda žinutę apie pakitimus (žinutė turėtų būti trumpa (ne ilgesnė nei 50 simbolių) ir aiški)
  • restore - sugrąžina failą į buvusio commit'o versiją (panaikina visus pakitimus darytus po paskutinio commit'o)
  • push - nusiunčia lokalios repozitorijos commit'us į išorinę repozitoriją
  • pull - atsiunčia išorinės repozitorijos commit'us į lokalią repozitoriją
  • clone - klonuoja nurodytą repozitoriją iš išorinės repozitorijos
  • log - išveda commit'ų sąrašą
  • checkout commit_ID|branchName - pereina į nurodyto kodo versiją arba kitą šaką
  • branch naujosSakosPavadinimas - sukuria naują šaką
  • ...

Branch protection rules

Vienas svarbiausių šakų apsaugos taisyklių yra "Pull Request" (bandymas vienos šakos commit'us sukelti į kitą šaką).
Pull Request dar galima suteikti "Require approvals" nustatyma, kuris reikalauja, kad nurodytas kiekis kitų projekto dalyvių patvirtintų Pull Request'ą.

Eiliškumas Pull Request

  1. Susikurti naują šaką - git branch newBranchName
  2. Nueiti į tą naują šaką - git checkout newBranchName
  3. Atlieki užduotį, kurią pasiėmei daryti per vieną ar kelis commit'us:
    • Add'inti pakitimus - git add .
    • Commit'ini pakitimus - git commit -m 'žinutė apie pakitimus'
  4. Nueiti į main|master šaką - git checkout main|master
  5. Atnaujini main|master šaką - git pull
  6. Grįžti į savo užduoties šaką - git checkout newBranchName
  7. Sujungti savo šaką su main|master šaka ir išspręsti galimai atsiradusius konfliktus - git merge main|master
  8. Nusiunti savo darbinės šakos pakitimus į gitHub'ą - git push
  9. GitHub'e padarai Pull Request'ą
  10. Kiti komandos nariai patvirtina arba atmeta Pull Request'ą
