Interface vs Type

Matt Pocock: TypeScript: Should you use Types or Interfaces?
Web Dev Simplified: I Cannot Believe TypeScript Recommends You Do This!

Interface gali būti naudojamas kuriant klasių arba objektų aprašymą.
Type gali būti naudojamas kuriant betkokį aprašymą.

Vienas Interface gali būti pratęstas kitu Interface naudojant 'extends' prierašą.
Vienas Type gali būti pratęstas kitu Type naudojant '&' (intersection) ženkliuką.

Interface gali būti papildytas naujomis savybėmis/metodais, negrįžtant prie to Interface aprašymo vietos, o per naują jį aprašant.
Type gali būti papildytas naujomis savybėmis/metodais/reikšmėmis..., bet norint tai atlikti būtina sugrįžti prie tos vietos, kur jis originaliai sukurtas.

Kuriant Interface negali nurodyti vieno arba kito varianto. Reikia susikurti du Interface, su vienu ir kitu variantu ir kuriant duomenį naudoti vieną arba kitą Interface.
Kuriant Type galima nurodyti vieną arba kitą variantą.

Išvada (mano asmeninė (Rokas Banaitis)): Siūlyčiau naudoti Type vietoj Interface. Nes Type gali atlikti beveik viską ką atlieką Interface, o taip pat daug dalykų, kurių Interface negali atlikti.
