TS Config

TypeScript konfigūracijos failas yra skirtas nurodyti kaip .ts tipo failai, turi būti kompiliuojami į .js tipo failus.
Config failo nustatymų yra labai daug, čia aptarsime keletą jų:

  • target - į kurią JS versiją kodą versti
  • rootDir - kuriame aplankale yra .ts failai, kuriuos reikia konvertuoti į .js failus
  • outDir - kuriame aplankale sudėti į .js kovertuotus .ts failus
  • noImplicitAny - neleidžia naudoti "any" tipo
  • noUnusedLocals - draustų nepanaudotus kintamuosius
  • noUnusedParameters - draustų nepanaudotus parametrus
  • noImplicitReturns - aprašyti funkcijos grąžinimą
  • removeComments - sukompiliuoja be komentarų
  • skipLibCheck - ar praleisti tipų tikrinimą naudojant bibliotekas
  • exactOptionalPropertyTypes - visos objekto savybės privalo būti aprašyti
  • forceConsistentCasingInFileNames - versti failus vadinti pagal vienodą struktūrą
  • strict - ar kodas privalo būti griežtos/tvarkingos sintaksės
  • alwaysStrict - sukompiliuoti .js failai turėtų "use strict"
  • noImplicitThis - "this" privalo turėti tipą
  • noFallthroughCasesInSwitch - switch cases turi turėti break'us

Masyvų tipai

Tuples

Tuple yra nurodyto ilgio masyvas, kurio reikšmių tipas ir eiliškumas svarbus. Kitaip pasakius Tuple yra griežtos struktūros masyvas.

["Rokas", "Banaitis"] - tuple.
{ vardas: "Rokas", pavarde: "Banaitis" } - object.
Tuple yra mažiau vietos užimantis, greičiau kodo nuskaitomas, bet mažiau programuotojui aiškus duomuo. Dėl to JS'e gan retai pasitaikantis.

Array

Kuriant masyvo tipą, reikia nusirodyti masyvo viduje esančius tipus ir masyvo gylį

Objektų tipai

Object

Galime objektą aprašyti paprastai, kaip kad tai darytume JS'e, bet negalime patys nurodyti tipų. Tipai ir automatiškai priskiriami kai nurodome reikšmes.
Dėl to yra naudojami type arba interface norint aprašyti objektus.

Type

Naudojamas norint sukurti objektų / masyvų / klasių... struktūras.
Norint sukurti tipą, reikia parašyti žodį "type" ir už jo pavadinimą ir struktūrą.
Savo sukurto tipo priskyrimas visiškai nesiskiria nuo jau egzistuojančių tipų priskyrimo.

Interface

Naudojamas norint sukurti objektų arba klasių struktūras.
Visą kitą labai panašu/identiška Type'ui.
