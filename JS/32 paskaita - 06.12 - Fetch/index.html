API

Application Programming Interface - jūsų užklausa iš aplikacijos (front-end) išsiunčiama į serverį (back-end) ir jeigu viskas tvarkoje - grąžinami kažkokie duomenys pagal jūsų užklausą; jeigu kažkas negerai - grąžinama error žinutė.

JSON

JSON - Java Script Object Notation yra duomenų formatas, kuris naudojamas request'uose.
JSON formatas nuo JS Object formato skiriasi tuo, kad: raktiniai žodžiai ir string reikšmės privalo būti dvigubose kabutėse IR po paskutinės reikšmės negali būti kablelio.

Tam, kad paversti JSON reikšmę į JS arba atvirkščiai, reikia naudoti JSON metodus:
1) JSON.parse(JSON) - paverčia JSON sintaksę į JS sintaksę 2) JSON.stringify(JS) - paverčia JS sintaksę į JSON sintaksę

Fetch

Funkcija/metodas, kurios pagalba mes galime siūsti užklausas į serverį ir gauti kažkokius atsakymus.

Fetch vykdomo eiga:

  1. Fetch'inant (siūnčiant užklausą) nurodomas URL, galima nurodyti papildomas opcija. Tarp tų opcijų yra metodai:
    • GET - gauti info (default)
    • POST - pateikti info (sukurti naują duomenį)
    • DELETE - trinti info (pagal ID)
    • PUT - redaguoti info (pagal ID ir info) (perrašo viską apart ID)
    • PATCH - redaguoti info (pagal ID ir info) (pakoreguoja/prideda)
  2. Apdoroti gautą info/atsakymą
  3. Naudoti gautą apdorotą data
  4. Pagauti (catch) error ir kažką su juo padaryti
  5. Galiausiai (finally) dar kažką atlikti (nesvarbu buvo ar nebuvo error)
