Semantika

Semantika yra reikalinga tam, kad puslapis būtų tvarkingas, lengva jame naviguotis (programuotojui) ir būtų geriau vertinamas google paieškos sistemos.

  • header - puslapio viršutinei daliai
  • nav - sekcija su navigacijos nuorodomis
  • aside - šoninė pagalbinė (nav,search,filter,sort...) juosta
  • main - pagrindinė puslapio informacija
  • article - main tag'o viduje esanti sekcija skirta vienai iš temų aprašyti
  • section - main tag'o viduje esanti sekcija skirta vienai iš temų aprašyti
  • footer - puslapio apatinė dalis

Naršyklės numatytieji stiliai

Kiekvienas (beveik) tag'as turi savo numatytuosius naršyklės stilius.
Skirtingos naršyklės gali turėti minimaliai skirtingus numatytuosius stilius tiems patiems tag'ams.
Tam, kad tag'ų stiliai būtų vienodi arba netgi panaikinti, yra naudojami Reset arb Normalize CSS'ai.

Reset CSS - suvienodina/panaikina visų tag'ų stilius visose naršyklėse. (asmeniškai nesiūlau naudoti)

Normalize CSS - suvienodina vienodų tag'ų stilius tarp skirtingų naršyklių, panaikina nenaudingus stilius ir prideda reikalingus stilius.

Tėviniai ir vaikiniai elementai

Elementas gali turėti DAUG vaikinių elementų.
Elementas TURI tik vieną tėvinį elementą.

Paveldimumas

Stiliai gali būti paveldimi iš tėvinio elemento į vaikinį elementą.
