Git ir Github

Git - programa, kuri versijuoja kodą.

Github - svetainė, kurioje keliamas (versijuotas) kodas.

Git setup

  1. Įsirašyti Git programą
  2. Pasitikrinti ar tikrai įrašytas Git'as. Terminale panaudoti komandą "git -v".
  3. Susikurti Github paskyrą.
  4. Konfigūruoti git'ą globaliai (jūsų naudojamame įrenginyje)
    • git config --global user.name "GithubUserName"
    • git config --global user.email "GithubEmail"
    • git config --global core.editor "code"
    • git config --global core.autocrlf true|input (true jeigu windows | input jeigu mac|linux)
  5. Pasitikrinti ar teisingai atlikta konfigūracija. Terminale panaudoti komandą "git config --global --edit"

Libraries ir Frameworks

CSS Libraries/Frameworks - kažkieno kito parašytas CSS kodas (kuris nėra originalioje CSS versijoje), kuriuo tu gali naudotis savo projektuose.
Norint naudotis Libraries/Frameworks - reikia juos prijungti prie savo projekto naudojant CDN.
Dauguma CSS Libraries/Frameworks veikia tokiu principu, kad prijungus jį prie savo projekto - nudoji jame aprašytas klases.
Yra ir Classless Libraries, kur nereikia net priskirinėti klasių, o tiesiog rašyti HTML'ą.

Pros

  • Pagreitina/palengvina darbą
  • Nereikai/labai mažai rašyti CSS
  • Viskas aiškiai vienoje vietoje
  • Įdiegtas responsiveness
  • ...

Cons

  • Neunikalus dizainas
  • Daug perteklinio/nenaudojamo kodo

  • Pasimiršta CSS
  • Sudėtingiau pritaikyti savo stilius
  • Reikia laiko mokintis skirtingus Libraries
  • Nešvarus HTML dėl klasių pertekliaus
  • ...

Preprocessors

CSS with superpowers

Suteikia galimybę rašyti CSS'ą naujoviškesniu būdu. Jame galima lengvai ir patogiai kurti kintamuosius, nest'inti, rašyti: sąlgas, funkcijas, ciklus, mix'inus ir t.t.

Naudojant betkurį CSS preprocesorių, reikia parašytą kodą konvertuoti į paprastą CSS'ą, kurį jau prijungiame prie svetainės.
